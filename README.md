## 📖 About The Project

**AI Coding Doubt Solver** is a beginner-friendly web application that leverages local AI models to help programmers understand code better. Whether you're just starting your coding journey or need help debugging complex logic, this tool provides instant, intelligent assistance.

### Why This Project?
- 🎯 **Perfect for beginners** who struggle with code comprehension
- 🔒 **Privacy-focused** - Runs locally with Ollama, no data sent to cloud
- 💰 **Completely free** - No API costs, no subscriptions
- 🚀 **Fast responses** - Local AI model provides quick analysis
- 📚 **Educational** - Learn by understanding code line by line

### Use Cases
- 🧑‍🎓 Students learning programming
- 👨‍🏫 Teachers explaining code concepts
- 👨‍💻 Developers debugging their code
- 🔍 Anyone wanting to understand complex code

### 🚀 **Step-by-Step Installation**

#### **Step 1: Clone the Repository**

Open your terminal/command prompt and run:

```bash
# Clone the repository
git clone https://github.com/surya-hash-coder/ai-coding-doubt-solver.git

# Navigate to project folder
cd ai-coding-doubt-solver
Step 2: Install Backend Dependencies
bash
# Go to backend folder
cd backend

# Install Node.js dependencies
npm install
Step 3: Install Ollama and AI Model
Install Ollama:

OS	Command/Instructions
Windows	Download from ollama.ai and install
macOS	brew install ollama
Linux	curl -fsSL https://ollama.ai/install.sh | sh
Pull an AI Model (Choose based on your RAM):

bash
# For 4GB RAM (Recommended for most users)
ollama pull qwen2.5-coder:1.5b

# For 8GB+ RAM
ollama pull deepseek-coder:6.7b

# For very low RAM systems
ollama pull tinyllama
Verify Installation:

bash
# Check if Ollama is running
ollama list
Step 4: Configure Environment Variables
Create a .env file in the backend folder:

bash
# Create .env file (Windows)
echo PORT=3000 > .env
echo OLLAMA_URL=http://localhost:11434 >> .env
echo MODEL_NAME=qwen2.5-coder:1.5b >> .env
Or create manually with this content:

env
# Server Configuration
PORT=3000

# Ollama Configuration
OLLAMA_URL=http://localhost:11434

# AI Model (change based on your downloaded model)
MODEL_NAME=qwen2.5-coder:1.5b
Step 5: Start the Backend Server
bash
# From the backend folder
node server.js
You should see:

text
🚀 Server running on http://localhost:3000
🤖 Using model: qwen2.5-coder:1.5b
🔗 Ollama URL: http://localhost:11434
Step 6: Open the Frontend
Option A: Direct Open

Navigate to frontend/index.html in File Explorer

Double-click to open in browser

Option B: Using VS Code Live Server

bash
# Open the frontend folder in VS Code
code frontend

# Right-click index.html → "Open with Live Server"
Option C: Using Python HTTP Server

bash
cd frontend
python -m http.server 8000
# Then open http://localhost:8000

http

  "feature": "explain"
}

text
Error: model requires more system memory (5.5 GiB) than is available (4.0 GiB)

bash

# Restart backend
</details><details> <summary><b>🔴 "Cannot connect to Ollama"</b></summary>
Error:

text
Error: Cannot connect to Ollama. Make sure Ollama is running.
Solutions:

Start Ollama:

Windows: Look for Ollama in system tray

Mac: Check menu bar

Linux: Run ollama serve

Check if Ollama is running:

bash
ollama list
Verify Ollama API:

bash
curl http://localhost:11434/api/tags
</details><details> <summary><b>🔴 "Port 3000 already in use"</b></summary>
Error:

text
Error: listen EADDRINUSE: address already in use :::3000
Solution:

bash
# Change port in .env
PORT=3001

# Or kill the process using port 3000
# Windows:
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Mac/Linux:
lsof -i :3000
kill -9 <PID>
</details><details> <summary><b>🔴 "CORS error"</b></summary>
Error in browser console:

text
Access to fetch at 'http://localhost:3000/api/analyze' from origin 'http://127.0.0.1:5500' has been blocked by CORS policy
Solution:

Make sure backend has CORS enabled (our server.js includes app.use(cors()))

Use same protocol (http://localhost vs http://127.0.0.1)

</details><details> <summary><b>🔴 "Git push permission denied"</b></summary>

text

bash
# Generate SSH key
ssh-keygen -t ed25519 -C "your-email@example.com"

# View public key
type %USERPROFILE%\.ssh\id_ed25519.pub

# Add to GitHub: https://github.com/settings/keys

# Test connection
ssh -T git@github.com
